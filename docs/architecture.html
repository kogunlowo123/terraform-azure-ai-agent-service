Azure AI Agent Service Infrastructure Module

Azure AI / OpenAI

Executive Summary

Terraform module for provisioning the complete Azure AI Agent Service infrastructure. Deploys an Azure OpenAI Cognitive Account with GPT-4o model deployment, Azure AI Search for grounded retrieval, a Storage Account with file-upload and code-interpreter containers, Key Vault for secret management, Log Analytics and Application Insights for observability, a User-Assigned Managed Identity with fine-grained RBAC role assignments, diagnostic settings, and optional private endpoints for network isolation across all services.

Data Flow

1
Client application calls Azure OpenAI Assistants API via SDK or REST.
2
OpenAI Cognitive Account processes the request using the GPT-4o deployment.
3
For RAG, OpenAI queries Azure AI Search for grounded retrieval using Search Index Data Reader role.
4
File uploads stored in the file-uploads blob container; code interpreter uses dedicated container.
5
API keys stored in Key Vault; managed identity accesses secrets via RBAC.
6
Diagnostic settings send audit logs and metrics to Log Analytics workspace.
7
Application Insights provides end-to-end tracing and performance monitoring.

Production Readiness Checklist

  • Enable private endpoints for all services
  • Configure IP range restrictions on OpenAI
  • Enable Application Insights monitoring
  • Set appropriate model capacity (TPM) for load
  • Configure storage replication type (GRS for production)
  • Enable Key Vault purge protection
  • Set log retention to compliance requirements
  • Provision in region with OpenAI model availability
  • Configure network ACLs to deny public access
  • Review RBAC role assignments for least privilege

Deployment

# Verify OpenAI access
az cognitiveservices account list-kinds | grep OpenAI

# Initialize and deploy
terraform init
terraform plan -out=tfplan
terraform apply tfplan

# Verify deployment
az cognitiveservices account show \
  --name myagent-openai \
  --resource-group rg-ai-agents \
  --query "properties.endpoint" -o tsv
